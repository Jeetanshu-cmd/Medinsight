Core Capabilities

Everything you need to modernize clinical data

From document ingestion to compliance reporting, MedInsight AI supports the full clinical data lifecycle.

Ingestion

Patient Record Ingestion Endpoint

`POST /api/ingest-report` receives a lab report or scan, extracts values via Claude, and writes structured fields into `patient_records`.

AI Engine

Claude Replaces Manual Entry

Clinical documents are parsed automatically so coordinators do not retype source values into case report workflows.

Validation

Protocol Rule Engine

After save, extracted values are compared against protocol thresholds and breaches are written into a `deviations` table with severity and timestamps.

Compliance

Audit Log Middleware

Every upload, extraction, edit, flag, and sign-off also creates an `audit_log` row with `user_id`, `record_id`, `timestamp`, and a SHA-256 `data_hash`.

Realtime

Supabase Realtime Alerts

New deviation rows broadcast over WebSocket automatically, so the frontend can surface red alerts instantly without extra backend plumbing.

Operations

Structured Reviewer Workspace

Teams review records, deviations, and approvals in one place with searchable records and a clear operational audit trail.

Workflow

From upload to insight in minutes

Four steps to turn unstructured clinical documents into validated, reviewable data.

01

Upload Report

A site submits a lab report or scan to the ingestion endpoint.

02

Extract Fields

Claude reads the document and fills structured patient fields automatically.

03

Run Rules

The validation engine compares values against protocol thresholds and writes deviations.

04

Broadcast Alerts

Supabase Realtime pushes new deviations to reviewers and dashboards immediately.
